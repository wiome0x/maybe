## 币安（Binance）配置与使用

### 1. 获取 API Key

1. 登录 [Binance](https://www.binance.com) → 右上角头像 → **API Management**
2. 点击 **Create API** → 选择 **System generated**
3. 权限只需勾选 **Read Only**（无需提现权限）
4. 保存生成的 **API Key** 和 **Secret Key**（Secret 只显示一次）

### 2. 在 Maybe 中连接

币安凭证是**用户级别**的，直接在账户创建/设置界面填写，**不需要**在 `.env` 里配置全局密钥。

流程：
1. 创建新的 **Crypto** 账户
2. 选择"连接币安（可选）"
3. 填入 API Key 和 Secret Key → 提交时会实时验证有效性
4. 验证通过后加密存入数据库，后续自动同步

### 3. SDK

使用 Binance 官方 Ruby SDK：[`binance-connector-ruby`](https://github.com/binance/binance-connector-ruby) v1.7.1

```ruby
# Gemfile
gem "binance-connector-ruby"

# 使用方式
client = Binance::Spot.new(key: api_key, secret: api_secret, timeout: 10)
client.account          # GET /api/v3/account
client.my_trades(symbol: "BTCUSDT", startTime: 1234567890000)
```

SDK 负责：HMAC-SHA256 签名、timestamp 注入、`X-MBX-APIKEY` header、错误分类（`Binance::ClientError` / `Binance::ServerError`）。

所有 SDK 调用统一经过 `call()` 包装器，确保每次 HTTP 交互都写入 `ApiRequestLog`。

## 嘉信理财（Charles Schwab）配置与使用

### 1. 注册开发者应用

1. 前往 [Schwab Developer Portal](https://developer.schwab.com)
2. 注册账号 → 创建应用 → 获取 **Client ID** 和 **Client Secret**
3. 在应用设置里填写 Callback URL：`https://你的域名/auth/schwab/callback`

### 2. 配置环境变量

嘉信使用 OAuth 2.0，需要在 `.env` / `.env.local` 里配置**应用级**密钥：

```bash
# Charles Schwab OAuth
SCHWAB_CLIENT_ID=你的_client_id
SCHWAB_CLIENT_SECRET=你的_client_secret
SCHWAB_REDIRECT_URI=https://你的域名/auth/schwab/callback
```

本地开发时填入 `.env.local`，生产环境填入 `.env.production` 或服务器环境变量。

### 3. 完善 `exchange_code` 实现

`Provider::Schwab.exchange_code` 目前是 TODO，需要根据嘉信文档补全：

```ruby
def self.exchange_code(code:)
  uri = URI(TOKEN_URL)
  http = Net::HTTP.new(uri.host, uri.port)
  http.use_ssl = true

  request = Net::HTTP::Post.new(uri)
  request["Content-Type"] = "application/x-www-form-urlencoded"
  # Basic Auth: Base64(client_id:client_secret)
  request["Authorization"] = "Basic #{Base64.strict_encode64("#{ENV['SCHWAB_CLIENT_ID']}:#{ENV['SCHWAB_CLIENT_SECRET']}")}"
  request.body = URI.encode_www_form(
    grant_type: "authorization_code",
    code: code,
    redirect_uri: ENV["SCHWAB_REDIRECT_URI"]
  )

  response = http.request(request)
  body = JSON.parse(response.body)

  {
    access_token:     body["access_token"],
    refresh_token:    body["refresh_token"],
    expires_in:       body["expires_in"],
    broker_account_id: nil  # 需要再调一次 /accounts 接口获取
  }
end
```

### 4. 在 Maybe 中连接

1. 创建新的 **Investment** 账户
2. 点击"连接嘉信理财（可选）"→ 跳转到嘉信授权页面
3. 用户登录嘉信并授权 → 回调到 `/auth/schwab/callback`
4. 自动换取 token 并创建 `BrokerConnection`

## 数据同步机制

### 连接模式

币安使用**短连接 REST 轮询**，不是长连接。每次 sync 发出若干 HTTPS 请求后立即关闭，没有 WebSocket 或持久连接。

### 触发路径

同步由两条独立路径触发，互为补充：

**路径 1 — 用户登录驱动**（`AutoSync` before_action）

```
用户访问任意页面
  └─ family_needs_auto_sync?
       条件：有活跃账户 && 今天未同步 && auto_sync_on_login = true
         └─ family.sync_later
              └─ Family::Syncer#child_syncables
                   ├─ PlaidItem × N        → PlaidItem::Syncer
                   └─ BrokerConnection × N → BrokerConnection::Syncer
                        ├─ GET /api/v3/account
                        ├─ GET /api/v3/myTrades × N（每个持仓 symbol）
                        └─ account.sync_later（子 sync，重算余额）
```

每天最多触发一次，Plaid 和 Binance 同时同步。

**路径 2 — 每日 cron**（`ScheduledBrokerSyncJob`，06:00 UTC）

```
ScheduledBrokerSyncJob
  └─ 仅同步 status=active 的 BrokerConnection
       └─ BrokerConnection::Syncer（同上）
```

只同步 BrokerConnection，**不触发 Plaid**。目的是补偿 Binance 没有 webhook 推送的缺陷，确保用户当天未登录时数据仍能更新。

### 安全与审计保障

| 维度 | 机制 |
|---|---|
| 密钥安全 | API Key/Secret 加密存储；`key`/`secret`/`signature` 等字段在写入审计日志前脱敏 |
| SDK 签名 | 由 `binance-connector-ruby` 官方处理，不再手写 HMAC-SHA256 |
| 每次 HTTP 调用入库 | `call()` 包装器在成功和失败时都调用 `log_http_request`，写入 `ApiRequestLog` |
| 审计字段 | `endpoint`（含 symbol）、`response_code`、`response_time_ms`、脱敏 `request_payload` / `response_payload` / `error_payload` |
| 完整追溯链 | 每条审计记录关联 `broker_connection_id` + `sync_id` |
| 错误不漏记 | `Binance::ClientError` 在 `call()` 里先入库再 raise，不存在吞掉错误的情况 |
| 防超频 | `guard_sync_rate!` 限制同一连接每天最多同步 10 次，防止触发 Binance 限流 |
| 失败处理 | auth 错误 → `requires_reauth`；其他错误 → `error`；超 24h 未完成 → `stale` |

**审计日志查询示例：**

```ruby
# 某个连接的所有 API 调用
ApiRequestLog.for_connection("broker_connection_id").recent

# 某次 sync 的所有调用
ApiRequestLog.for_sync("sync_id").recent

# 最近的错误
ApiRequestLog.errors.by_provider("binance").recent.limit(20)
```

## 总结

| | 币安 | 嘉信 |
|---|---|---|
| 密钥类型 | 用户 API Key/Secret | OAuth Client ID/Secret |
| 配置位置 | 用户在界面填写，加密存 DB | `.env` 里配置应用级密钥 |
| 需要 `.env` 变量 | 无 | `SCHWAB_CLIENT_ID` `SCHWAB_CLIENT_SECRET` `SCHWAB_REDIRECT_URI` |
| 认证方式 | HMAC-SHA256 签名（由官方 SDK 处理） | OAuth 2.0 授权码流程 |
| 同步方式 | REST 轮询（登录驱动 + 每日 cron） | REST 轮询（登录驱动，无 cron） |
| 数据延迟 | 秒级实时 | 秒级实时 |
