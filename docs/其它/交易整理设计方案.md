# 交易整理设计方案

## 背景与定位

`交易整理` 是家庭共享账户中的数据整理与分析入口，目标是提升交易数据的可读性、一致性和可分析性，服务后续的搜索、筛选、聚合、趋势分析与研究复盘。

当前产品虽然具备预算、分类、规则、导入、自动识别等能力，但在整体定位上，不再将“预算驱动型家庭记账”作为该场景的主叙事，而是转向“家庭共享容器下的交易整理与分析”。

在这一定位下：

- 家庭用于共享账户、共享视图和共同维护交易数据
- 交易整理用于提升数据质量和分析口径
- 后续报表、搜索、规则、筛选都依赖整理结果
- 预算能力保留，但不作为该模块的主要入口逻辑

## 核心结论

当前阶段保留 `分类`、`交易方`、`标签` 三个独立模型，不做业务合并。

原因如下：

- `分类` 是主分析口径，直接影响统计、报表和聚合结果
- `交易方` 是对手方或商户识别对象，适合做名称归并、搜索和规则命中
- `标签` 是横向补充维度，适合承载多选场景信息

三者职责不同，若合并会导致数据语义混乱：

- `分类` 与 `标签` 合并后，会混淆“唯一主归类”和“多选补充信息”
- `交易方` 与 `标签` 合并后，会混淆“稳定识别对象”和“灵活场景标记”
- `交易方` 与 `分类` 合并后，会混淆“交易对象”和“分析口径”

因此当前阶段的产品策略是：

- 数据模型保持独立
- 页面入口未来可统一
- 语义边界必须清晰

## 信息架构

未来产品层采用一个统一的 `交易整理` 总入口，下分三个分区：

- `分类`
- `交易方`
- `标签`

该总入口负责承载：

- 说明三者的角色差异
- 提供统一的整理指引
- 提供常用示例与命名建议
- 承接后续规则、批量处理和自动识别能力

当前阶段不直接开放这一总入口，而是先完成语义收口与文档沉淀。

## 角色定义

### 分类

`分类` 表示“这笔交易属于什么分析口径”。

适用范围：

- 消费分类，如餐饮、交通、家庭用品
- 收入分类，如工资、股息、利息
- 投资相关分类，如手续费、税费、转账、收益归类

规则：

- 一笔交易通常只有一个主分类
- 分类优先服务统计、报表、趋势和分析
- 分类不承担补充备注职责

### 交易方

`交易方` 表示“这笔交易和谁发生”。

适用范围：

- 商户，如星巴克、Costco、京东
- 金融机构，如招商银行、盈透证券、Charles Schwab
- 公司或平台，如工资发放公司、订阅服务平台

规则：

- 一笔交易通常只有一个交易方
- 交易方优先服务搜索、聚合、自动识别和规则命中
- 交易方不承担分析口径职责

### 标签

`标签` 表示“这笔交易还有哪些横向特征”。

适用范围：

- 家庭语义，如家庭、孩子、父母
- 事务语义，如报销、订阅、税务相关
- 研究语义，如长线、波段、观察、复盘、高股息

规则：

- 一笔交易可以有多个标签
- 标签用于补充场景，不替代分类
- 标签优先服务多维筛选、专题分析和复盘视图

## 家庭与投资场景示例

### 家庭消费场景

示例：在 Costco 采购家庭日用品

- 分类：`家庭用品`
- 交易方：`Costco`
- 标签：`家庭`

说明：

- `家庭用品` 用于报表统计
- `Costco` 用于交易方聚合
- `家庭` 用于横向筛选家庭共同开销

### 家庭事务场景

示例：支付孩子培训班费用

- 分类：`教育`
- 交易方：`XX 培训机构`
- 标签：`孩子`

说明：

- 家庭成员归属当前阶段先通过标签表达
- 暂不新增成员归属专用字段

### 投资平台场景

示例：券商收取月度市场数据费

- 分类：`手续费`
- 交易方：`Interactive Brokers`
- 标签：`订阅`

说明：

- `手续费` 作为主分析口径
- `Interactive Brokers` 用于平台聚合
- `订阅` 用于识别周期性研究成本

### 投资研究场景

示例：围绕一笔研究型交易做后续复盘

- 分类：按交易主类型归类
- 交易方：对应券商或平台
- 标签：`观察`、`成长`、`复盘`

说明：

- 分类保持稳定
- 研究语义通过标签补充，避免污染主分类体系

## 默认执行规则

当前阶段的执行顺序统一为：

1. 先识别 `交易方`
2. 再归类 `分类`
3. 最后补充 `标签`

对应原则如下：

- 导入或同步后，优先识别稳定的交易对象
- 分类用于建立统一分析口径
- 标签用于补充附加语义，不反向替代分类
- 规则系统优先围绕交易方和分类建立稳定命中
- 标签更适合用于手动补充或规则批量打标

## 当前阶段产品策略

由于 `分类`、`交易方`、`标签` 三者的入口体验、页面叙事和整体组织方式尚未完全定型，当前阶段采取以下策略：

- 保留现有底层模型、路由和页面实现
- 暂时从设置导航中隐藏 `分类`、`交易方`、`标签` 三个独立入口
- 不影响直接访问原页面地址
- 不影响交易编辑页、批量编辑、规则、导入映射和已有自动识别能力
- 后续待统一入口 `交易整理` 设计完成后，再重新组织对外展示方式

这一策略的目的，是先统一语义和设计方向，再进行交互层整合，避免用户在当前阶段将三个页面理解为彼此独立且并列的最终方案。
